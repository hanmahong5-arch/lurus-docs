# Webmail 常见问题

## 登录问题

### 忘记密码怎么办？

联系管理员重置：admin@lurus.cn，说明你的邮箱地址即可。重置后会收到临时密码邮件，**请在 24 小时内登录并修改**。

### 账号被锁定了？

连续 5 次密码错误会触发 15 分钟锁定，等待后自动解除。紧急情况联系管理员手动解锁。

### Web 界面无法访问？

1. 确认网络可以访问 [mail.lurus.cn](https://mail.lurus.cn)
2. 清除浏览器缓存（`Ctrl+Shift+Del`）或尝试无痕模式
3. 若仍无法访问，联系 support@lurus.cn

## 发送问题

### 给 QQ / 163 / 126 邮箱发邮件对方收不到？

国内主流邮箱通过 SendCloud 中继发送，偶有延迟。请按以下顺序排查：

**第一步：让对方检查垃圾邮件文件夹**，大多数情况问题到此解决。

**第二步：让对方将 `@lurus.cn` 加入白名单：**

:::tabs
== QQ 邮箱
1. 登录 QQ 邮箱 → **设置** → **反垃圾**
2. 点击「**添加白名单**」→ 输入 `@lurus.cn`
3. 点击保存

== 163 / 126 邮箱
1. 登录 → **设置** → **反垃圾/黑白名单**
2. 选择「**白名单**」→「**添加白名单**」
3. 输入 `lurus.cn`，选择「域名」类型，保存

== Outlook / Hotmail
1. 登录 Outlook.com → **设置**（⚙️）→ **垃圾邮件**
2. 「**安全发件人和域**」→「**添加**」
3. 输入 `lurus.cn`，点击保存

== Gmail
1. 打开该邮件 → 右上角「⋮」→「**筛选此类邮件**」
2. 点击「**创建过滤条件**」→ 勾选「**永不发送到垃圾邮件**」
3. 点击保存
:::

**第三步：** 等待 5–10 分钟后重试。若仍未送达，联系 support@lurus.cn 并提供：收件方邮箱域名 + 发送时间。

### 发邮件报错 "Relay denied"？

原因：SMTP 认证失败。检查：

- 用户名必须是**完整邮箱地址**（如 `zhangsan@lurus.cn`，不能只写 `zhangsan`）
- 密码是否正确（区分大小写）
- 端口和加密是否匹配（推荐 `587` / STARTTLS）

### 附件发送失败？

- 单封邮件附件上限 **25 MB**，超过请改用网盘分享链接
- 附件文件名避免特殊字符（`* / \ : ? " < > |`）

### 邮件被对方标为垃圾邮件？

我们已配置 SPF / DKIM / DMARC。如果仍被拦截，可以自行验证邮件的认证状态：

```bash
# 查看邮件头部的认证结果（在对方那里查看原始邮件头）
# 正常情况应看到：
Authentication-Results: mx.google.com;
   dkim=pass header.d=lurus.cn;
   spf=pass smtp.mailfrom=lurus.cn;
   dmarc=pass

# 或用命令行验证 DNS 记录
dig TXT lurus.cn | grep spf
# 应返回：v=spf1 include:sendcloud.net ~all

dig TXT mail._domainkey.lurus.cn
# 应返回 DKIM 公钥（v=DKIM1; k=rsa; p=...）
```

如果上述记录异常，联系 support@lurus.cn 提供完整邮件头信息。

## 接收问题

### 客户端收不到新邮件？

1. 确认 IMAP 连接参数正确：服务器 `mail.lurus.cn`，端口 `993`，SSL/TLS
2. 手动触发「检查邮件」或「刷新」
3. 检查邮件是否被过滤规则移到其他文件夹

### 邮件跑到垃圾箱了？

1. 打开垃圾邮件文件夹，找到该邮件
2. 点击「**标记为非垃圾邮件**」
3. 将发件人加入白名单：「**设置**」→「**白名单**」→「**添加发件人**」

## 客户端配置问题

### Outlook 报错"无法连接到服务器"？

请严格按以下参数配置：

| 类型 | 服务器 | 端口 | 加密 |
|------|--------|------|------|
| IMAP（收件） | mail.lurus.cn | 993 | SSL/TLS |
| SMTP（发件） | mail.lurus.cn | 587 | STARTTLS |

::: warning
用户名必须填写**完整邮箱地址**，包含 `@lurus.cn`。部分客户端默认只填用户名前缀会导致认证失败。
:::

### iPhone 配置后发件显示失败？

iOS 邮件 App 有时对 STARTTLS（587 端口）有兼容性问题。改用 SMTP 端口 `465` + `SSL/TLS` 加密通常可以解决。

### Thunderbird 提示证书错误？

1. 检查系统时间是否准确（证书验证依赖系统时间，误差超过 5 分钟会报错）
2. 删除现有账户重新添加
3. 若出现「证书不受信任」提示，检查证书颁发机构是否为 Let's Encrypt，选择「永久信任」

## 存储与配额

### 邮箱存储配额是多少？

默认每个账号 **10 GB**。查看已用空间：Web 界面右下角，或「**设置**」→「**账户信息**」。

### 如何清理空间？

| 操作 | 节省空间 |
|------|---------|
| 清空垃圾箱和垃圾邮件 | 通常可释放 30–50% |
| 删除含大附件的旧邮件 | 过滤「附件大于 5 MB」搜索定位 |
| 清空「已发送」中的大附件邮件 | 已发送通常占用最多空间 |

::: tip 快速搜索大附件
在搜索框输入 `has:attachment larger:5MB` 可快速定位大邮件。
:::

### 配额不够怎么申请扩容？

联系管理员（admin@lurus.cn）申请，说明你的邮箱地址和需要的容量。

## 安全

### 如何开启两步验证？

Webmail 通过 Zitadel 统一认证，在 [auth.lurus.cn](https://auth.lurus.cn) → **账号安全** 中开启 TOTP（推荐 Google Authenticator 或 1Password）。

### 如何查看登录记录？

「**设置**」→「**安全**」→「**登录历史**」，查看近 30 天的登录 IP 和时间。

::: danger 发现异常登录
立即：① 修改密码 ② 撤销所有其他登录会话 ③ 联系 admin@lurus.cn 报告异常 IP
:::
